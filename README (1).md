# RTL JOULES OPT TOOLS
**8-bit Universal Shift Register — PPAC Analysis using Verilog HDL and Vivado**

This project designs, verifies, and compares two RTL implementations of an 8-bit Universal
Shift Register on an Artix-7 FPGA. The goal is to study **PPAC (Power, Performance, Area,
Congestion)** trade-offs between an intentionally non-optimized design and an
optimization-aware design, inspired by industrial Joules RTL Design Studio methodology.

## Design Summary

| Feature | Non-Optimized | Optimized |
|---|---|---|
| Always block style | Separate combo + clocked | Single clocked block |
| Reset type | Active-high synchronous | Active-low asynchronous |
| Reset FPGA mapping | D-input mux (wastes LUT) | Dedicated FF CLR pin |
| Clock enable | None (FF always active) | FF CE pin (no LUT) |
| Serial outputs | Registered (2 extra FFs) | Wire taps (zero FF delay) |
| Intermediate nodes | WIDTH `d` nodes | None |
| Expected timing @ 100 MHz | **FAILS (negative WNS)** | **PASSES (positive WNS)** |
| Expected dynamic power | Higher | Lower |
| Expected area | More LUTs + FFs | Fewer LUTs + FFs |

> **Parameterisation:** Both RTL modules use `parameter WIDTH = 8`. The testbench derives all
> structural constants (`NON_OPT_FF = WIDTH + 2`, `OPT_FF = WIDTH`, `NON_OPT_NODES = WIDTH`)
> automatically — changing `WIDTH` to 16 or 32 requires zero manual updates.

**Mode encoding** (same for both designs):

| `mode[1:0]` | Operation |
|---|---|
| `2'b00` | Hold |
| `2'b01` | Shift Right (`ser_in_r` → MSB) |
| `2'b10` | Shift Left (`ser_in_l` → LSB) |
| `2'b11` | Parallel Load |

## Simulation (Vivado xsim)

### Step 1 — Compile
```bash
xvlog universal_shift_register_8bit_non_opt.v
xvlog universal_shift_register_8bit_opt.v
xvlog tb_universal_shift_register.v
```

### Step 2 — Elaborate
```bash
xelab -debug typical tb_universal_shift_register -s tb_sim
```

### Step 3 — Simulate
```bash
xsim tb_sim -runall
```

The testbench prints a full PPAC comparison report to the console.
It also generates `sim_dump.vcd` for VCD-based power analysis (see below).

A reference copy of the expected console output is committed at [`sim_output.log`](sim_output.log).

### Expected Console Output (summary)
```
  FUNCTIONAL SUMMARY
  NON-OPT : Passed=9   Failed=0
  OPT     : Passed=9   Failed=0

  OVERALL PPAC SUMMARY
  Performance  : NON-OPT FAILS 100MHz  | OPT PASSES 100MHz
  Power        : NON-OPT Higher switching | OPT Lower switching
  Area         : NON-OPT More LUTs + FFs  | OPT Fewer LUTs + FFs
  Congestion   : NON-OPT Deeper netlist   | OPT Compact netlist
```

## Synthesis and Implementation (Vivado — Automated)

Run the full flow with one command from the Vivado Tcl Console or batch mode:

```bash
vivado -mode batch -source run_all.tcl
```

This script:
1. Creates an in-memory project targeting `xc7a35tcpg236-1`
2. Runs synthesis, opt_design, place_design, route_design
3. Generates timing, utilization, and power reports in `reports/`
4. Repeats the same flow for the optimized design

### Manual GUI Flow (alternative)
1. Create a new RTL project targeting **xc7a35tcpg236-1**
2. Add the relevant `.v` source and `.xdc` constraint file
3. Run **Synthesis** → **Implementation**
4. Open **Report Timing Summary**, **Report Utilization**, **Report Power**
5. Save reports to the `reports/` folder

Expected results:

| Design | WNS | LUTs | FFs | Dynamic Power |
|---|---|---|---|---|
| Non-optimized | Negative (FAIL) | More | 10 | Higher |
| Optimized | Positive (PASS) | Fewer | 8 | Lower |

## VCD-Based Power Analysis (Vivado Tcl)

After simulation, `sim_dump.vcd` is generated. To perform exact mW power analysis:

```tcl
# In Vivado Tcl Console (after implementation of opt design)
read_vcd sim_dump.vcd -strip_path tb_universal_shift_register/dut_opt
report_power -file reports/power_vcd_opt.rpt
```

This gives exact mW dynamic power based on actual switching activity from simulation,
replacing the toggle-count proxy approach with Vivado's internal power model.

## Scaling to 16-bit / 32-bit

Both RTL modules and the testbench are fully parameterised. To study PPAC at wider widths:

```verilog
// In testbench: change parameter WIDTH = 8  →  16 or 32
parameter WIDTH = 16;
```

All structural constants (`NON_OPT_FF`, `OPT_FF`, `NON_OPT_NODES`) and stress test patterns
adjust automatically. Re-run synthesis to compare area and timing at the wider width.
